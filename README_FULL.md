AIG Optimization Experiments — Full README
=======================================

Overview
--------

This repository contains a small prototype for measuring how logic optimizations (using Berkeley ABC) affect internal node correspondence between an original BLIF network and optimized variants.

The goal is to answer: after each optimization, how many internal nodes still match exactly, and when exact matching fails, do we still see similar supports/regions?

Repository structure
--------------------

```
aig_optimization_experiments/
  benchmarks/                 small hand-written BLIF examples
  variants/                   generated optimized BLIF files (created by run_abc_variants.sh)
  logs/                       ABC logs for each variant run
  results/                    CSV results and plots generated by analysis
    summary_metrics.csv       per-benchmark/optimization summary metrics
    top_candidates.csv        top-K candidate matches for each optimized node
    plots/                    PNG plots (node counts, support overlap)
  run_abc_variants.sh         runs ABC optimizations and writes variants/
  analyze_blif_matches.py    computes matching metrics and top-k candidates
  visualize_results.py       generates plots from results/summary_metrics.csv
  README.md                  short README (kept)
  README_FULL.md             this comprehensive README
```

How to reproduce locally
------------------------

Requirements:
- Python 3.10+ (script uses list[str] hints but will run on Python 3.8+ mostly)
- Berkeley ABC (https://github.com/berkeley-abc/abc) — needs make and a C compiler
- pip packages for visualization: `pandas`, `matplotlib` (only if you want to regenerate plots)

Steps:

1. Build ABC (if you don't have it):

```bash
git clone https://github.com/berkeley-abc/abc.git
cd abc
make
# this creates an `abc` binary in the repo root
```

2. Generate optimized BLIF variants:

```bash
# from this repository root
ABC=/path/to/abc ./run_abc_variants.sh
```

3. Run the analysis script:

```bash
python3 analyze_blif_matches.py
```

4. Generate plots (optional):

```bash
python3 -m pip install --user pandas matplotlib
python3 visualize_results.py
```

Produced outputs
----------------

- `results/summary_metrics.csv` — columns: benchmark, optimization, original_nodes, optimized_nodes, original_levels, optimized_levels, exact_internal_matches, old_signatures_disappeared, new_signatures_appeared, avg_best_support_overlap, simulation_mode

- `results/plots/` — per-benchmark node-count plots and a support_overlap plot

Interpreting the current results
--------------------------------

- On tiny benchmarks like `toy_and_or` and `mux2`, ABC's transformations preserved node-level signatures for most optimizations.
- The `xor_chain` benchmark under the `resyn2_like` flow lost exact node-level matches (all signatures changed) while keeping an average support overlap of ~0.81 — this is the important case motivating region-level matching.
